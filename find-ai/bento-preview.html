Design notes

Why this works for Find.ai

🎨 Apple-style soft

24px rounded corners, soft shadows (4% opacity), muted pastel accents (blue/yellow/green/red/purple) against navy + white. No aggressive gradients, no startup neon.

📐 Mixed tile sizes

Hero gets 2-row height. Stats and quick actions stay 1-row. Pastel topic tiles always paired. The eye lands on the biggest tile first — that's where the primary action lives.

🇲🇾 Malaysian property context

Every pastel tile represents a real topic from your knowledge base: stamp duty, deposit, eviction, foreign buyer. No generic "AI features" — every tile = a legal situation users actually have.

🔐 Trust signals throughout

PDPA-compliant chip, pulsing live dot, "Malaysia-trained" tagline. Keeps the Mature Minimalism / bank-level trust vibe from your CLAUDE.md.

📱 Mobile-first

All three screens shown at 390px wide (iPhone). Bento collapses gracefully — no awkward 3-across at narrow widths. Desktop can use wider grids (4-6 cols).

🔁 Reuses existing code

Same Tailwind palette, same EN/BM/中文 translations, same starter questions data. This is a layout refactor, not a rewrite — ~2 hours of work, not a week.

⚠️ Risks to flag
  • • Translation work: ~15 new tile labels × 3 languages = 45 strings to write. Manageable but real.
  • • Returning users: muscle memory break for ~1 week. Expect "where did X go?" feedback.
  • • Feature tile discoverability: if too many topics, becomes a choice-paralysis grid. Cap at 6-8.
  • • Chat MID-STREAM still stays linear (icon cards). Bento is only for landing / empty-state / history. Answers themselves don't change.
